Bytes

Average request bytes is an average size of all incoming messages from the satellites. During idle time, this is just a small "is there work?" message, but during build times this is a full log of events. Average response bytes represents the status messages returned to the satellites, and is typically small and consistent.

Idle

Idle is the percentage of time the central server stands doing nothing. Even when builds are in progress, central does little, with most of the work being done by the satellites.

Ping

Ping is the millisecond round-trip response time incurred when the 'status' command is run. As the server is single-threaded, this reflects load.

Service

Service time represents the average time to process a request from a client, in seconds. As the server is single-threaded, this reflects load. The max service time is a high-water mark that can reflect central server downtime.

Total Bytes

Total bytes in and out in the form of requests and responses. This is a cumulative quantity that is reset on central restart.

TPS

Transactions per second.

Transactions

Transaction count is a cumulative quantity that is reset on central restart.
