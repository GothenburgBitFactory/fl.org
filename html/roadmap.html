IDEAS

This list represents features and suggestions. It is unsorted and makes no attempt to identify cost, feasibility or utility.

  • Automatically tagging clean commits (hooks)
  • Automatically generate dev tarballs from clean commits (hooks)
  • Central status dashboard
  • Project status widget
  • Project commit backfill
  • Specifiable root dir
  • Commit rebuild
  • Dependency enumeration
  • Revocable security tokens
  • Calculate and display build times
  • Support non-default make targets
  • Support MAKEFLAGS
  • Satellite 'nice' for builds
  • Satellite clone proxy, cleared on idle
  • Work timeout based on statistics
  • Ignore list for file types, so a README change doesn't force a build
  • Support svn
  • Support hg
  • Support bzr
  • Support autoconf
  • Support configure

  • Automatic naming/iding/fingerprinting of satellites
  • Generalized hook system, with user-defined events
  • Timeout for lengthy/frozen build/test cycles

FUTURE

  • SSL communication, optional
