Central Node

The central node is a server. It is typically run continuously on a machine that is available in terms of uptime and connectivity. It can be run either as a daemon, or simply a long-running process. Its purpose is to coordinate the build/test effort, and provide analysis. There is typically one central node supporting multiple projects.

Data Storage

The central server needs a work directory which is used to store all the transcripts, for every platform, every commit, every branch of every project. While projects and branches can be locked to prevent testing, this data accumulates and can grow to become large.

Every notification of a commit results in the creation of a node in the work tree. Every build/test result is stored within that node. The structure of the node is simple: 

work
└── <project>
    └── <branch>
        ├── <commit>
        │   ├── build.msg
        │   ├── ...
        └── tinderbox.html

The 'build.msg' file shown represents the build request, which is a notification from the repository. This includes all the necessary details to perform the test. The 'tinderbox.html' file shown is a test report, which is generated per branch.
